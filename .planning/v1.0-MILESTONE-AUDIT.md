## Integration Check

**Status:** Complete — 12 connections wired, 2 critical gaps, 1 orphaned export

### Critical Wiring Gaps

#### 1. `syncQueueRepo.enqueue()` — 0 call sites (SYNC-02, SYNC-03)

The sync infrastructure is complete (queue repo, flush logic, privacy guard, retry tracking) but **nothing ever writes to the queue**. No repo or store calls `syncQueueRepo.enqueue()` after data writes. The entire sync pipeline is dead — `flushQueue()` on app foreground always processes 0 items.

**Impact:** Authenticated users' SYNCABLE data (habits, XP, quests, titles, settings) is never backed up to Supabase.

#### 2. `authStore.userId` not consumed by tabs (SYNC-01 integration)

`authStore.userId` correctly updates to the real Supabase UUID on `SIGNED_IN`, and `migrateGuestData()` correctly re-keys SQLite rows. However, all 5 tab screens (`habits.tsx`, `index.tsx`, `quests.tsx`, `profile.tsx`, `your-data.tsx`) hardcode `'default-user'` as the userId constant.

**Impact:** After "Keep My Progress" auth flow, migration moves data to real UUID, but tabs query with `'default-user'` and see empty state. User loses all visible data after signing in.

**Files:** `app/(tabs)/habits.tsx:38`, `app/(tabs)/index.tsx:29`, `app/(tabs)/quests.tsx:71`, `app/(tabs)/profile.tsx:32`, `app/your-data.tsx:24`
