    - buildMessagePayload({ buddyPairId: 'bp1', senderId: 'u1', content: 'hi' }) returns object with id (UUID string), status='queued', createdAt (ISO string), syncedAt=null
    - buildMessagePayload sets status to 'queued' (not 'sent') for local-first approach
    - MAX_MESSAGE_LENGTH equals 1000
    - MESSAGE_STATUSES contains 'queued', 'sending', 'sent', 'filtered', 'failed'

  <action>
    RED: Write test file __tests__/domain/message-engine.test.ts with all behavior cases above. Run tests -- all must fail.

    GREEN: Create src/domain/message-engine.ts:
    - Export const MAX_MESSAGE_LENGTH = 1000
    - Export const MESSAGE_STATUSES = ['queued', 'sending', 'sent', 'filtered', 'failed'] as const
    - Export type MessageStatus = typeof MESSAGE_STATUSES[number]

    - initProfanityFilter(): void -- initializes leo-profanity with custom Islamic blocklist. Uses a boolean flag to ensure idempotency. Start with minimal Islamic blocklist (empty array is fine -- per STATE.md blocker, this must be hand-curated).

    - validateMessageContent(content: string): { valid: true } | { valid: false; reason: 'empty' | 'too_long' | 'profanity' }

      2. If trimmed length > MAX_MESSAGE_LENGTH, return { valid: false, reason: 'too_long' }
      3. If !isClean(trimmed), return { valid: false, reason: 'profanity' }
      4. Return { valid: true }

      id: crypto.randomUUID() (or uuid() helper if project uses one -- check imports)

      status: 'queued'
      createdAt: new Date().toISOString()
      syncedAt: null
