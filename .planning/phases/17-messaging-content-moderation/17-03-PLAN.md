  <name>Task 1: Create messageStore with send, receive, report, and Realtime subscription</name>
  <files>src/stores/messageStore.ts</files>
  <action>
    Create src/stores/messageStore.ts following duoQuestStore pattern (Zustand, no persist middleware):

    Interface:
    ```typescript
    interface MessageState {
      messages: Message[];
      loading: boolean;
      sending: boolean;
      activeBuddyPairId: string | null;
      error: string | null;

      loadMessages: (buddyPairId: string, limit?: number, offset?: number) => Promise<void>;
      loadMore: () => Promise<void>;
      sendMessage: (buddyPairId: string, senderId: string, content: string) => Promise<{ ok: boolean; reason?: string }>;
      receiveMessage: (message: Message) => void;
      reportMessage: (messageId: string, reporterId: string, reason: string) => Promise<void>;
      subscribeToRealtime: (buddyPairId: string) => () => void;
      clearMessages: () => void;
    }
    ```

    Implementation details:

    loadMessages(buddyPairId, limit=50, offset=0):
    - Set loading=true, activeBuddyPairId=buddyPairId
    - Call messageRepo.getByBuddyPair(buddyPairId, limit, offset)
    - Set messages to result (descending from repo, UI will use inverted FlatList)
    - Set loading=false

    loadMore():
    - If no activeBuddyPairId or loading, return
    - Call messageRepo.getByBuddyPair with offset = current messages.length
    - Append older messages to end of messages array

    sendMessage(buddyPairId, senderId, content):
    - Call validateMessageContent(content) from message-engine
    - If invalid, return { ok: false, reason }
    - Call buildMessagePayload({ buddyPairId, senderId, content })
    - Call messageRepo.create(payload) to write locally with status='queued'
    - Prepend to messages array (newest first for inverted FlatList)
    - Set sending=true
    - If supabaseConfigured: call supabase.functions.invoke('moderate-message', { body: { buddyPairId, senderId, content, localId: payload.id } })
      - If response ok: update local status to 'sent' via messageRepo.updateStatus, update in messages array
      - If response filtered: update local status to 'filtered', return { ok: false, reason: 'content_filtered' }
      - If response error/network fail: leave as 'queued' (message-queue.ts will retry)
    - If !supabaseConfigured: leave as 'queued' (offline mode)
    - Set sending=false
    - Return { ok: true } on success

    receiveMessage(message):
    - Only add if message.buddyPairId matches activeBuddyPairId
    - Only add if message not already in messages array (deduplicate by id)
    - Also write to local SQLite via messageRepo.create (cache for offline viewing)
    - Prepend to messages array

    reportMessage(messageId, reporterId, reason):
    - Call messageRepo.reportMessage({ id: crypto.randomUUID(), messageId, reporterId, reason, createdAt: new Date().toISOString() })

    subscribeToRealtime(buddyPairId):
    - If !supabaseConfigured, return no-op cleanup function
    - Create Supabase channel: supabase.channel(`messages:${buddyPairId}`)
    - Subscribe to postgres_changes: event='INSERT', schema='public', table='messages', filter=`buddy_pair_id=eq.${buddyPairId}`
    - On new message: call receiveMessage(payload.new) -- but skip if senderId is current user (we already have it locally)
    - Return cleanup function that calls supabase.removeChannel(channel)

    clearMessages():
    - Reset messages=[], activeBuddyPairId=null, error=null

  <name>Task 2: Create dedicated message queue flush service for offline messages</name>
  <files>src/services/message-queue.ts</files>
  <action>
    Create src/services/message-queue.ts -- a dedicated flush function for queued messages that routes through the Edge Function (NOT the generic sync engine). This is critical: offline messages MUST go through server-side moderation per MSSG-02/MSSG-06.

    ```typescript
    import { messageRepo } from '../db/repos/messageRepo';
    import { supabase, supabaseConfigured } from '../lib/supabase';
    import NetInfo from '@react-native-community/netinfo';
    ```

    flushMessageQueue():
    - If !supabaseConfigured, return (no server to send to)
    - Get queued messages: messageRepo.getQueuedMessages() (already ordered by createdAt ASC)
    - For each message in order (MSSG-06: "in order"):
      - Update status to 'sending' via messageRepo.updateStatus
      - Call supabase.functions.invoke('moderate-message', { body: { buddyPairId, senderId, content, localId: message.id } })
      - If ok: update status to 'sent'
      - If filtered: update status to 'filtered'
      - If network error: break loop (stop flushing, will retry on next connectivity event)
    - Process sequentially (not Promise.all) to maintain order

    startMessageQueueListener():
    - Subscribe to NetInfo.addEventListener
    - When connectivity changes to isConnected=true, call flushMessageQueue()
    - Return unsubscribe function for cleanup

    IMPORTANT: This service runs INSTEAD of the generic sync engine for messages. The generic sync engine should never process messages entityType. Since messageRepo.create does not enqueue to syncQueueRepo, this is already handled by design.
