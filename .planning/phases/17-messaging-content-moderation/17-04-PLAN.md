    Create supabase/functions/moderate-message/index.ts following the push function pattern (Deno runtime):

    ```typescript
    import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';
    ```

    Server-side blocklist (simple string array -- no npm packages in Deno Edge Functions):
    ```typescript
    const SERVER_BLOCKLIST: string[] = [
      // Common English profanity (hand-curated subset)
      // Start conservative -- can expand server-side without app update
      // Per STATE.md blocker: Islamic content blocklist must be hand-curated
    ];

    function containsProfanity(text: string): boolean {
      const lower = text.toLowerCase();
      return SERVER_BLOCKLIST.some(word => {
        // Word boundary check to avoid false positives on substrings
        const regex = new RegExp(`\\b${word}\\b`, 'i');
        return regex.test(lower);
      });
    }
    ```

    Main handler flow (Deno.serve):
    1. Only accept POST requests. Return 405 for other methods.

    2. Extract and verify JWT from Authorization header:
       - Create a Supabase client with SUPABASE_URL + SUPABASE_ANON_KEY (not service role) to verify the JWT
       - Call supabase.auth.getUser(token) to validate and extract user ID
       - If invalid: return { ok: false, reason: 'unauthorized' }

    3. Parse request body: { buddyPairId, content, localId }
       - Extract senderId from the verified JWT (do NOT trust client-supplied senderId -- research Pitfall #4)

    4. Verify sender is a member of the buddy pair:
       - Create a service-role client (SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY)
       - Query buddies table: select where id = buddyPairId AND (user_a = senderId OR user_b = senderId) AND status = 'accepted'
       - If no match: return { ok: false, reason: 'not_member' }

    5. Check content against server blocklist:
       - If containsProfanity(content): return { ok: false, reason: 'content_filtered' }
       - Rejected messages are NEVER inserted into the database (MSSG-02)

    6. Insert message using service-role client:
       ```typescript
       const { data, error } = await serviceClient
         .from('messages')
         .insert({
           id: crypto.randomUUID(),
           buddy_pair_id: buddyPairId,
           sender_id: senderId,
           content,
           status: 'sent',
           created_at: new Date().toISOString(),
         })
         .select()
         .single();
       ```

    7. Return { ok: true, message: data }

    Error handling:
    - Wrap entire handler in try/catch
    - Return 500 with { error: string } on unexpected errors
    - Log errors to console for Supabase function logs

    CORS headers:
    - Add Access-Control-Allow-Origin, Allow-Methods (POST, OPTIONS), Allow-Headers (authorization, content-type)
    - Handle OPTIONS preflight request

    Security notes:
    - NEVER trust client-supplied senderId -- always extract from JWT
    - SERVICE_ROLE_KEY bypasses RLS, so buddy pair membership check is mandatory
    - Server blocklist can be updated by redeploying the function (no app update needed)
