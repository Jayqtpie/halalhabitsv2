    First, install react-native-worklets:
    ```bash
    npx expo install react-native-worklets
    ```
    Do NOT add any babel plugin manually -- babel-preset-expo handles it automatically.

    Create XPFloatLabel (src/components/game/XPFloatLabel.tsx):
    - Floating text that rises from a given position and fades out
    - Uses Reanimated useSharedValue + useAnimatedStyle (RESEARCH Pattern 5)
    - Text format: "{baseXP} x {multiplier}x = +{cappedXP} XP" (per user decision -- show CAPPED amount, not earned)
    - Press Start 2P font, emerald/jewel-tone color (colors.dark.primary or similar accent)
    - Animation: translateY from 0 to -80 over 1200ms, opacity fades from 1 to 0 starting at 600ms
    - Positioned absolutely over the habit card using measureInWindow (not measure -- per Pitfall 3)
    - Calls onDone callback when animation completes via runOnJS
    - Renders in a portal-like overlay at root level (not inside FlatList) to avoid scroll position issues

    Create XPProgressBar (src/components/game/XPProgressBar.tsx):
    - Persistent bar showing level progress
    - Header line: "Lv {level} . {activeTitle?.name}" in pixel font (Press Start 2P) -- title only shown if activeTitle exists
    - Bar: horizontal fill bar with emerald fill color, dark track
    - Label: "{currentXPInLevel} / {xpForNextLevel} XP" right-aligned or below bar
    - Animated fill using useSharedValue (RESEARCH Pattern 6):
      - Normal XP gain: withTiming to new progress, 800ms, Easing.out(cubic)
      - On level-up: withSequence(fill to 100% in 400ms, hold 200ms, instant reset to 0, fill to new progress in 600ms)
    - Reads totalXP and currentLevel from useGameStore
    - Computes progress: (totalXP - xpForLevel(currentLevel)) / xpForNextLevel(currentLevel)
    - Height: ~8-10px bar with 4px border radius, padded in spacing.md horizontal

    Modify habits screen (app/(tabs)/habits.tsx):
    - Import and render XPProgressBar below the DailyProgressBar component
    - Add XPFloatLabel rendering logic:
      - When lastXPResult changes to non-null, capture the card position and show the float
      - Use a ref to track the last-completed card's position (pass via onComplete callback from HabitList/HabitCard)
      - Float label renders in an absolute overlay View at the SafeAreaView level
    - Add level/title display in the header area:
      - Between "Today" header and DailyProgressBar, or integrated into XPProgressBar header
    - Call gameStore.loadGame(DEFAULT_USER_ID) in useEffect alongside loadDailyState

    IMPORTANT per Pitfall 3: Use measureInWindow (not measure) for card position. The float label must be positioned at root SafeAreaView level, not inside the FlatList.

    IMPORTANT: Do NOT read shared values during render (Pitfall from RESEARCH anti-patterns). Only read in useAnimatedStyle or worklets.
