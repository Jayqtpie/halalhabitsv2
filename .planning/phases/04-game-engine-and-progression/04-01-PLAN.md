  <files>src/domain/xp-engine.ts, __tests__/domain/xp-engine.test.ts</files>
  <behavior>
    - xpForLevel(1) returns 0 (starting level)
    - xpForLevel(2) returns 40 (first threshold)
    - xpForLevel(5) returns ~540 (achievable week 1)
    - xpForLevel(10) returns ~2900
    - xpForLevel(20) returns ~13000
    - xpForLevel(100) returns ~610000
    - xpForNextLevel(1) returns 40 (gap from level 1 to 2)
    - levelForXP(0) returns 1
    - levelForXP(39) returns 1
    - levelForXP(40) returns 2
    - levelForXP(540) returns 5
    - levelForXP(999999) returns 100 (capped)
    - applySoftCap(100, 0) returns 100 (below cap, no reduction)
    - applySoftCap(100, 450) returns 75 (50 below cap at full + 50 above at 50%)
    - applySoftCap(100, 500) returns 50 (all above cap, 50% reduction)
    - applySoftCap(100, 600) returns 50 (already past cap)
    - calculateXP(15, 1.5, 0, 0) returns { baseXP: 15, multiplier: 1.5, earnedXP: 22, cappedXP: 22, ... }
    - calculateXP with enough XP to level up sets didLevelUp: true and correct newLevel
    - calculateXP with dailyTotal near cap applies soft cap correctly
    - XP economy simulation: at 5 habits/day (avg 25 base XP, 1.5x multiplier), level 5 reachable in ~4 days
  </behavior>
  <implementation>
    Follow the exact architecture from RESEARCH.md Pattern 1. Use formula: 40 * Math.pow(level - 1, 1.85). Binary search for levelForXP. Soft cap at 500 XP daily with 50% diminishing returns above cap. Export XPResult interface.

    CRITICAL: The XP float label will show cappedXP (not earnedXP). The cap is invisible to the user -- no mention of cap in any UI-facing data.
  </implementation>
