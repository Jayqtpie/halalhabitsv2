    - unlockedTitleIds: Set of string (for fast lookup)
    - activeQuests: Quest[] (current active quests)

    Actions:
    - loadGame(userId): Load user record, compute level from xpRepo.getLifetimeTotal (source of truth reconciliation per Pitfall 4), load user titles, load active title details, load active quests. Set all state.
    - awardXP(userId, baseXP, multiplier, sourceType, sourceId): Call calculateXP from xp-engine. Write to xpRepo.create. Update userRepo.updateXP. Update local state. Check for title unlocks via checkTitles. Return XPResult.
    - checkTitles(userId, stats): Load all title conditions from titleRepo.getAll(). Build alreadyUnlocked Set from unlockedTitleIds. Call checkTitleUnlocks from title-engine. For each newly unlocked title, call titleRepo.grantTitle. Update local state. Set pendingCelebration if any unlocked.
    - updateQuestProgress(userId, event): For each active quest, call evaluateQuestProgress. If progress increased, call questRepo.updateProgress (use atomic SQL increment pattern: SET progress = MIN(progress + 1, target_value) to avoid race condition per Pitfall 5). If progress >= targetValue, call questRepo.complete and award bonus XP via awardXP.
    - generateQuests(userId, playerLevel): Expire old quests. Check slot counts. Call selectQuestTemplates for each type needing slots. Create quest records via questRepo.create. Update activeQuests state.
    - equipTitle(userId, titleId): Call userRepo.setActiveTitle. Load title details. Update activeTitle state.
    - dismissCelebration(): Set pendingCelebration to null.

    CRITICAL for awardXP: XP ledger is source of truth. If level derived from ledger sum differs from users.currentLevel, reconcile by recomputing from getLifetimeTotal.

    CRITICAL for pendingCelebration: If a title unlocks during a level-up (didLevelUp && newTitles.length > 0), bundle into a single 'level_up' celebration with unlockedTitles array. Never queue two celebrations simultaneously.

      6. Call useGameStore.getState().updateQuestProgress(userId, event)
    - Store the XPResult on the store state as lastXPResult (so UI can show the float label)
    - Add lastXPResult: XPResult | null to HabitState interface
    - Clear lastXPResult after UI consumes it (or on next completion)
