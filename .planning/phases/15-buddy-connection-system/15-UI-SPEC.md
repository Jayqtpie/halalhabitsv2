| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Buddy card internal element gaps, list item vertical gaps |

| lg | 24px | Screen outer padding, modal outer padding |
| xl | 32px | Layout section gaps |
| xxl | 48px | Major section breaks (e.g., empty state to CTA) |

Component-level presets (from src/tokens/spacing.ts — componentSpacing):

| Preset | Value | Usage in this phase |
|--------|-------|---------------------|

## Screen Inventory

### S-01: Buddy List Screen (main tab screen)

**Route:** `app/(tabs)/buddies.tsx`

**Layout (top to bottom):**
1. **Search bar** — inline TextInput at top, placeholder "Search by username...", debounced 300ms
2. **Pending Requests section** (visual focal point when present) — conditional, only shows when pending > 0. Header: "Pending Requests ({count})". Each request card has "Accept Request" (emerald) and "Decline Request" (gray) buttons.
3. **My Buddies section** (visual focal point when no pending) — FlatList of BuddyCard components. Header: "My Buddies ({count}/20)"
4. **FAB area** — Two buttons at bottom: "Invite a Buddy" (primary CTA) and "Enter Code" (secondary)

**Empty state** (when no buddies and no pending):
- Pixel-art illustration of two characters (centered, 120x120px placeholder)
- Heading (headingXl): "Accountability starts with one connection"
- Body (bodyMd): "Invite a friend or enter their code to get started"
- Two CTAs: "Invite a Buddy" (emerald filled) + "Enter Code" (emerald outlined)

### S-02: Buddy Profile Screen (push navigation)

**Route:** `app/buddy-profile/[id].tsx`

**Layout (top to bottom):**
1. **Header** — Back arrow, buddy username (headingMd), three-dot menu (Remove, Block) with accessibilityLabel "Buddy actions menu"
2. **Avatar section** — Large avatar circle (80x80px) with first-letter fallback, centered
3. **Identity Title** — PressStart2P 12px, centered below avatar (e.g., "The Steadfast")
4. **Stats grid** — 2x2 grid of stat cards on surface background:
   - Level (with level number)
   - XP Total (gold #FFD700)
   - Current Streak (with flame icon)
   - Days Connected (buddy tenure)
5. **Privacy notice** — bodySm, opacity 0.6: "Only public progress is shared"

### S-03: Invite Code Modal (bottom sheet)

**Trigger:** Tap "Invite a Buddy" on buddy list

**Layout:**
1. **Heading** (headingMd): "Share Your Invite Code"
2. **Code display** — Large monospace text (headingXl, 28px), centered: "HH-A7K3"
3. **Expiry label** (bodySm): "Expires in 47h 32m"
4. **Share button** — Full-width emerald CTA: "Share Code" → opens native share sheet
5. **Copy fallback** — Text link below: "or tap code to copy"

### S-04: Enter Code Modal (bottom sheet)

**Trigger:** Tap "Enter Code" on buddy list

**Layout:**
1. **Heading** (headingMd): "Enter Buddy Code"
2. **TextInput** — Centered, large monospace, auto-uppercase, max 7 chars (HH-XXXX format)
3. **Send Request button** — Full-width emerald CTA: "Send Request" (disabled until valid format)
4. **Error states** — bodySm, destructive color:
   - Not found: "Code not found — check for typos and try again"
   - Expired: "Code has expired — ask your buddy to share a new invite"

### S-05: Discoverability Prompt (one-time bottom sheet)

**Trigger:** First visit to buddy tab (if discoverability not yet set)

**Layout:**
1. **Heading** (headingMd): "Want others to find you?"
2. **Body** (bodyMd): "When enabled, other players can search for you by username. You can change this anytime in Settings."
3. **Two buttons:**
   - "Enable Search" (emerald filled)
   - "Keep Private" (gray outlined)

## Interaction Patterns

| Interaction | Behavior |
|-------------|----------|
| Tap buddy card | Push navigate to buddy profile screen |
| Long-press buddy card | No action (avoid accidental triggers) |
| Swipe buddy card | No swipe actions (use three-dot menu instead for remove/block) |
| Pull-to-refresh buddy list | Refresh buddy list and online statuses from Supabase |
| Tap "Accept Request" | Accept request, card animates from pending to buddy list |
| Tap "Decline Request" | Fade out card, remove from pending section |
| Tap "Remove" in menu | Show ConfirmActionSheet, remove on confirm |
| Tap "Block" in menu | Show ConfirmActionSheet (destructive styling), block on confirm |
| Search input typing | Debounce 300ms, query Supabase for discoverable users |
| Share code | Open native share sheet with text: "Join me on HalalHabits! Use code: HH-XXXX" |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Touch targets | All interactive elements minimum 44x44px |
| Screen reader | BuddyCard: "{username}, {online/offline}, streak {N}" |
| Screen reader | Accept/Decline: "Accept buddy request from {username}" / "Decline buddy request from {username}" |
| Screen reader | Three-dot menu: accessibilityLabel "Buddy actions menu" |
| Color contrast | Online dot green-500 on surface-800: 4.5:1 minimum |
| Reduce motion | Card transitions use `LayoutAnimation` with `UIManager.setLayoutAnimationEnabledExperimental` only when reduce motion is off |
