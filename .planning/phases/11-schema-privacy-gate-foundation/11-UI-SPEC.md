# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: Schema & Privacy Gate Foundation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification: No-UI Phase

Phase 11 is explicitly a **data-layer-only phase**. The CONTEXT.md states: "No feature UI or game logic — just the data layer foundation that Phases 12-17 build on." The RESEARCH.md confirms: "Phase 11 is a pure data-layer phase: no UI, no feature logic."

**What this phase produces:**
1. TypeScript source files (schema.ts edits, privacy-gate.ts edits, common.ts edits)
2. SQL migration files (Supabase RLS policies)
3. Test files (Jest test extensions)
4. One Markdown document (XP economy model)

**What this phase does NOT produce:**
- No new screens
- No new React Native components
- No new navigation routes
- No user-facing copy beyond enum string values

All tokens below are inherited from the existing design system in `src/tokens/`. They are recorded here to give downstream consumers a single reference point, not to establish new decisions.

## Spacing Scale

Inherited from `src/tokens/spacing.ts`. No new tokens needed for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, list item gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, modal padding offset |
| xl | 32px | Layout gaps |
| xxl | 48px | Major section breaks |

Component spacing presets (from `src/tokens/spacing.ts — componentSpacing`):

| Preset | Value | Usage | Justification |
|--------|-------|-------|---------------|
| cardPadding | 16px | All card containers | Standard md token |
| buttonPaddingVertical | 12px | All button heights | Between sm (8px) and md (16px); preserves 44px minimum touch target on standard text buttons without excessive height on compact UI elements. Established in Phase 2 and carried forward. |
| buttonPaddingHorizontal | 24px | All button widths | Standard lg token |
| modalPadding | 24px | Modal sheet containers | Standard lg token |
| hudCardPadding | 12px | HUD overlay cards | Between sm (8px) and md (16px); HUD cards are information-dense and use 12px to fit more content in the overlay without breaking the 8-point grid rhythm. Established in Phase 5 and carried forward. |

Exceptions: 44px minimum touch target for all interactive elements (ui-ux-pro-max rule: `touch-target-size`). This phase introduces no interactive elements, so exception is documented for downstream phases that reference these tables.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | colors.dark.background | #0F172A | App background, screen base |
| Secondary (30%) | colors.dark.surface | #1E293B | Cards, nav, list items, code blocks |
| Accent (10%) | colors.dark.primary | #0D7C3D | Reserved: primary CTAs, active states, border focus, XP bar fill |
| Destructive | colors.dark.error | #9B1B30 | Destructive actions only (block buddy, abandon battle, exit detox early) |

Accent (`#0D7C3D`) is reserved for: primary action buttons, active tab indicators, streak count highlights, and border focus rings. It must not be applied to informational labels or secondary text.

**Privacy classification comment colors in code:** No color tokens apply. Code comments use plain text. Privacy level annotations (`// Privacy: PRIVATE`, `// Privacy: SYNCABLE`, `// Privacy: LOCAL_ONLY`) use the established comment style from schema.ts — no visual treatment.

### Error State Copy (Privacy Gate)

These are runtime error messages thrown by assertSyncable(). They appear in logs and crash reports — not user-facing, but must be clear enough for a developer to diagnose.

| Trigger | Error Message |
|---------|--------------|
| Unknown table passed to assertSyncable() | `Privacy Gate: table '{tableName}' not classified in PRIVACY_MAP. Add an entry before syncing.` |
| PRIVATE table passed to sync queue | `Privacy Gate: table '{tableName}' is PRIVATE and must never be synced. Remove from sync queue call.` |
| LOCAL_ONLY table passed to sync queue | `Privacy Gate: table '{tableName}' is LOCAL_ONLY and must not be synced.` |

Note: These error messages follow existing patterns in `src/services/privacy-gate.ts`. If the existing messages use a different format, match the existing format exactly rather than overriding.

### No Destructive Actions in This Phase

Phase 11 introduces no destructive user interactions. The `abandoned` status for boss_battles and `exited_early` status for detox_sessions are data states — they have no confirmation dialogs in this phase. Those confirmation flows belong to Phase 13 (Detox) and Phase 14 (Boss).

## Phase Scope Reminder for Executor

This UI-SPEC is intentionally minimal because Phase 11 has no user-facing output. When implementing, the executor must:

1. Use existing token imports from `src/tokens/` — do not hardcode any hex values or spacing values inline.
2. Follow the Privacy classification comment format exactly as documented in the Copywriting Contract above.
3. Use the exact string literal values for status enums as listed in the Copywriting Contract — these become data stored in SQLite and must not drift across phases.
4. The XP economy document (`src/domain/xp-economy-v2.md`) is a technical artifact, not player copy. Plain Markdown, no design tokens.

Phases 12-17 will introduce user-facing screens. Each of those phases will have a UI-SPEC extending this token foundation with screen-specific contracts.
