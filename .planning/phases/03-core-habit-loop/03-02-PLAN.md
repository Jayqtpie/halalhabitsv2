StreakState interface (from research):
```typescript
export interface StreakState {
  currentCount: number;
  longestCount: number;
  multiplier: number;
  lastCompletedAt: string | null; // ISO 8601
  isRebuilt: boolean;
  mercyMode?: MercyModeState;
}

export interface MercyModeState {
  active: boolean;
  recoveryDay: number; // 0-3 (0 = not started, 3 = complete)
  preBreakStreak: number;
}
```

Key project decisions:
- Day boundary = midnight local time (00:00), not Fajr/Maghrib
- Multiplier: 1.0x base, +0.1x/day, cap 3.0x
- Mercy Mode is PER-HABIT (not global)
- Recovery: 3 consecutive days -> 25% of pre-break streak restored
- Fresh start: reset to 0, equally valid choice
- XP total never decreases (streak multiplier resets, not XP)

<feature>
  <name>Streak Engine</name>
  <files>src/domain/streak-engine.ts, __tests__/domain/streak-engine.test.ts</files>
  <behavior>
    processCompletion:
    - First completion: streak = 1, multiplier = 1.0
    - Consecutive day: streak + 1, multiplier + 0.1 (capped at 3.0)
    - Same day re-completion: no change (idempotent)
    - Non-consecutive (gap > 1 day): streak resets to 1, multiplier resets to 1.0
    - Updates longestCount if current exceeds it
    - During mercy recovery: increments recoveryDay

    detectStreakBreak:
    - Returns broken=true if lastCompletedAt is more than 1 calendar day before currentDate
    - Returns broken=false if lastCompletedAt is yesterday or today
    - Returns MercyModeState with preBreakStreak on break
    - Handles null lastCompletedAt (no prior completions, no break)

    processRecovery:
    - Day 1, 2: complete=false, restoredStreak=0
    - Day 3: complete=true, restoredStreak = floor(preBreakStreak * 0.25)
    - Minimum restored streak is 1 (if preBreakStreak >= 4)

    startFreshReset:
    - Returns StreakState with currentCount=0, multiplier=1.0, no mercy mode

    calculateStreakShieldBonus:
    - Returns true if completion time is within prayer window (start <= time < end)
    - Returns false if outside window
    - Only applicable to salah-type habits

    isCompletedToday:
    - Returns true if lastCompletedAt is same calendar day as reference date
    - Uses midnight local time as day boundary
  </behavior>
  <implementation>
    All functions are pure: (input) => output. No side effects, no DB, no React.
    Use ISO 8601 date strings throughout. Day comparison uses YYYY-MM-DD extraction.
    Helper: getHabitDay(isoString: string): string returns YYYY-MM-DD in local time.
    Helper: isNextDay(prev: string, current: string): boolean checks calendar day adjacency.
  </implementation>
</feature>
