  <action>
    Extend the existing habitStore to orchestrate the full completion flow. The store is the bridge between UI components and the data/domain layers.

    Add to HabitState interface:
    - completions: Record&lt;string, boolean&gt; (habitId -> completedToday, derived on loadDailyState)
    - streaks: Record&lt;string, StreakState&gt; (habitId -> streak state from streakRepo)
    - mercyModes: Record&lt;string, MercyModeState&gt; (habitId -> mercy mode state for active recoveries)
    - dailyProgress: { completed: number, total: number } (for "3 of 7" display)

    Add methods:
    - loadDailyState(userId: string, date?: string): Load active habits, today's completions, and streak states. Build the completions/streaks/mercyModes/dailyProgress derived state. Call habitRepo.getActive, completionRepo.getAllForDate (today), streakRepo.getAllForUser. For each habit, check if it has a completion today and detect streak breaks via detectStreakBreak from streak-engine.
    - completeHabit(habitId: string, userId: string):
      1. Check idempotency (already completed today? skip)
      2. Get current streak from streakRepo
      3. Call processCompletion from streak-engine to get new streak state
      4. Create completion record via completionRepo.create (xpEarned=0 for now, Phase 4 adds XP; streakMultiplier from streak state)
      5. Upsert streak via streakRepo.upsert
      6. If mercy mode was active and recovery completed, update accordingly
      7. Update local state (completions, streaks, dailyProgress)
    - startRecovery(habitId: string): Activate Mercy Mode recovery for a habit. Update streakRepo mercy mode columns. Set mercyModes[habitId].
    - resetStreak(habitId: string): Fresh start. Call streakRepo.resetStreak. Clear mercy mode. Update local state.
    - updateHabit(habitId: string, data: Partial): Call habitRepo.update, refresh habits list.
    - getHabitsForDisplay(): Return habits sorted via sortHabitsForDisplay from habit-sorter, enriched with completedToday, streak, and prayerWindow data (HabitWithStatus[]).

    Import from domain layers:
    - processCompletion, detectStreakBreak, processRecovery, startFreshReset from streak-engine
    - sortHabitsForDisplay from habit-sorter
    - completionRepo, streakRepo from repos

    Keep existing loadHabits, addHabit, archiveHabit methods. loadDailyState supersedes loadHabits for the daily view but loadHabits is kept for simpler use cases.

    Use generateId() from utils/uuid for new completion/streak record IDs.
    Use date utilities for day boundary calculations (midnight local time).
