**Step 2 — HudScene.tsx:**
Full-screen Skia Canvas rendering background PNG for current environment, scaled to fill screen. Structure:
```
Canvas (StyleSheet.absoluteFill)
  └── Background Image (current environment PNG, sampling: FilterMode.Nearest)
  └── DayNightOverlay (tint Rect)
  └── CharacterSprite (animated idle)
```

- Import `Canvas, Image, useImage, FilterMode, MipmapMode` from '@shopify/react-native-skia'
- Accept `level: number` prop, call `getEnvironmentForLevel(level)` to determine which background PNG to load
- Use a `const ENVIRONMENT_IMAGES` map from EnvironmentId → `require(...)` for the 4 environment PNGs
- Guard `if (!bg) return null;` — must null-check useImage result
- Use `Dimensions.get('window')` for screen width/height
- Render background with `fit="cover"` and `sampling={{ filter: FilterMode.Nearest, mipmap: MipmapMode.Nearest }}`

**Step 3 — DayNightOverlay.tsx:**
Skia Rect covering full screen with animated tint color based on local time:
- `getTimeProgress()` helper: `(hours * 60 + minutes) / (24 * 60)` → value 0-1
- `TIME_STOPS = [0, 0.22, 0.42, 0.75, 0.92, 1]`
- `TINT_COLORS = ['rgba(20,15,60,0.55)', 'rgba(255,180,80,0.20)', 'rgba(0,0,0,0.00)', 'rgba(255,100,40,0.25)', 'rgba(20,15,80,0.50)', 'rgba(20,15,60,0.55)']`
- Use `useSharedValue(getTimeProgress())` initialized with current time
- `useEffect` interval every 60000ms updating the shared value
- `useDerivedValue` calling `interpolateColors(timeProgress.value, TIME_STOPS, TINT_COLORS)` — import `interpolateColors` from '@shopify/react-native-skia'
- Render `<Rect x={0} y={0} width={screenW} height={screenH} color={tintColor} />`

**Step 4 — CharacterSprite.tsx:**
Animated 4-frame idle using manual frame cycling (simpler than Atlas for ≤4 frames):
- Accept `x: number, y: number` props for scene position
- `FRAME_W = 32, FRAME_H = 48, FRAME_COUNT = 4`
- `useImage(require('../../../assets/sprites/character-idle.png'))`
- `frame = useSharedValue(0)` driven by `withRepeat(withTiming(FRAME_COUNT, { duration: 800, easing: Easing.steps(FRAME_COUNT, false) }), -1, false)`
- `clipRect = useDerivedValue(() => ({ x: Math.floor(frame.value) * FRAME_W, y: 0, width: FRAME_W, height: FRAME_H }))`
- Render with Skia `Group` clip and translated `Image` — use pattern from RESEARCH.md Pattern 3 (simpler manual variant)
- Guard `if (!spriteSheet) return null;`

IMPORTANT: Do NOT apply animated `transform` on a Skia `Group` (Android performance regression per RESEARCH.md Pitfall 4). Animate x/y props directly or use the manual clip approach.

**EnvironmentReveal.tsx:**
Triggered when `gameStore.pendingCelebrations` contains a `level_up` celebration that crosses an environment boundary. Sequence: fade to black (400ms) → hold with "New Area Unlocked" text + environment name (800ms) → fade out (1200ms). Fires `Haptics.notificationAsync(NotificationFeedbackType.Success)` at reveal moment. After animation, calls `consumeCelebration()`.

Internal logic: compare `getEnvironmentForLevel(celebration.level - 1)` vs `getEnvironmentForLevel(celebration.level)` — only animate if different.

Uses `StyleSheet.absoluteFillObject` (same pattern as LevelUpOverlay.tsx from Phase 4 — reuse that pattern exactly, do NOT use React Navigation modal).

```typescript
export default function HomeScreen() {
  const { currentLevel, pendingCelebrations, consumeCelebration, loadGame } = useGameStore(useShallow(...));
  const router = useRouter();
  const { isAfterIsha } = useMuhasabahTrigger(); // derive from muhasabahStore + prayer-times

  useEffect(() => {
    loadGame('user-1'); // consistent with existing habits.tsx pattern
  }, []);

  return (
    <View style={StyleSheet.absoluteFill}>
      {/* Layer 1: Pixel art scene */}
      <HudScene level={currentLevel} />

      {/* Layer 2: Interactive tap zones */}
      <SceneObjects
        onTapQuestBoard={() => router.push('/(tabs)/quests')}
        onTapPrayerMat={() => router.push('/(tabs)/habits')}
        onTapJournal={() => useMuhasabahStore.getState().open()}

      />

      {/* Layer 3: HUD stat overlay */}
      <HudStatBar />

      {/* Layer 4: Celebration overlays */}
      <CelebrationManager />
      <EnvironmentReveal />
    </View>
  );
}
```
