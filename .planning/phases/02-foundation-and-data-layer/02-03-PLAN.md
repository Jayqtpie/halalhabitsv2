    - habitStore initializes with empty habits array and loading=false
    - gameStore initializes with level=1, totalXP=0, currentMultiplier=1.0, titles=[]
    - uiStore initializes with no active modals, no toasts
    - settingsStore initializes with defaults (prayerCalcMethod='ISNA', darkMode='auto', soundEnabled=true, hapticEnabled=true)
    - settingsStore persists via sqliteStorage
    - All stores use Zustand v5 patterns (useShallow for multi-field selectors)
  </behavior>
  <action>
    1. Create src/stores/sqliteStorage.ts:
       - Implements Zustand's `StateStorage` interface: `getItem(name)`, `setItem(name, value)`, `removeItem(name)`
       - Uses `getDb()` from src/db/client.ts to read/write the `_zustand_store` table
       - getItem: SELECT value FROM _zustand_store WHERE key = ?
       - setItem: INSERT OR REPLACE INTO _zustand_store (key, value) VALUES (?, ?)
       - removeItem: DELETE FROM _zustand_store WHERE key = ?
       - This is ~20 lines of code — thin adapter, NOT hand-rolling persistence

    2. Create src/stores/habitStore.ts:
       ```typescript
       interface HabitState {
         habits: Habit[];
         loading: boolean;
         error: string | null;
         loadHabits: (userId: string) => Promise<void>;
         addHabit: (habit: NewHabit) => Promise<void>;
         archiveHabit: (habitId: string) => Promise<void>;
       }
       ```
       - Uses `create` from zustand. NO persist (habit data is in SQLite via repos, not Zustand persist)
       - Actions call habitRepo methods, then update local state
       - Import { Habit, NewHabit } from types/database.ts

    3. Create src/stores/gameStore.ts:
       ```typescript
       interface GameState {
         currentLevel: number;
         totalXP: number;
         currentMultiplier: number;
         titles: UserTitle[];
         activeTitle: Title | null;
         loading: boolean;
       }
       ```
       - NO persist (game data is in SQLite via repos). Store is a read cache.

    4. Create src/stores/uiStore.ts:
       ```typescript
       interface UIState {
         activeModal: string | null;
         toastMessage: string | null;
         toastType: 'success' | 'error' | 'info' | null;
         showModal: (id: string) => void;
         hideModal: () => void;
         showToast: (message: string, type: 'success' | 'error' | 'info') => void;
         hideToast: () => void;
       }
       ```
       - NO persist (ephemeral UI state)

    5. Create src/stores/settingsStore.ts:
       - Uses `persist` middleware with `createJSONStorage(() => sqliteStorage)`
       - Default values: prayerCalcMethod='ISNA', darkMode='auto', soundEnabled=true, hapticEnabled=true, muhasabahReminderTime='21:00'
       - Partialize to persist only data fields, not action functions
       - This is the ONLY store that uses persist middleware (settings are user preferences)

    IMPORTANT: Use `useShallow` for any multi-field selector examples in comments. Zustand v5 requires this to prevent infinite re-renders.

    6. Write tests in __tests__/stores/stores.test.ts:
       - Test each store initializes with correct default values
       - Test settingsStore action methods update state correctly
       - Test uiStore modal show/hide works
       - Note: sqliteStorage adapter tests require expo-sqlite runtime, so mock the db for unit tests
  </action>

  </verify>
