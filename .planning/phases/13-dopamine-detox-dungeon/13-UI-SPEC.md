## Component Inventory

New components to build in Phase 13. Each maps to a decision in 13-CONTEXT.md.

### 1. `DungeonDoorIcon` (HUD overlay, Skia or RN Image)
- Pixel art dungeon gate icon, rendered at 32x32 source resolution
- Two states: `idle` (no glow) and `active` (emerald glow ring pulsing at 2s cycle)
- Touch target: minimum 44x44px hit slop via `hitSlop` prop
- During active session: shows countdown timer badge (pill) below icon using `hudLevel` font
- Source: `CONTEXT.md D-04`, `D-06`

### 2. `DungeonSheet` (bottom sheet modal)
- Full-width bottom sheet, `modalPadding: 24px` outer padding
- Header: Sheet title at `headingMd`, subtitle at `bodyMd`
- Daily/Deep toggle: two-segment control at top; "Daily" = default; "Deep" shows "ONCE PER WEEK" caption badge in `textMuted`
- Duration chips row: horizontal flex-wrap, chips at 44px minimum height, using `hudLabel` pixel font labels (2H, 4H, 6H, 8H)
  - Unselected: `surface-800` fill, `border-default` border
  - Selected: `rgba(13,124,61,0.15)` fill, `primary` border
  - Disabled (deep variant used this week): 40% opacity, non-interactive
- Primary CTA button: full-width, `buttonPaddingVertical: 12px`, `buttonPaddingHorizontal: 24px`, accent fill
- When session active: shows countdown timer, XP earned-so-far progress, "Exit Early" secondary destructive action
- Source: `CONTEXT.md D-01`, `D-02`, `D-03`

### 3. `DetoxCountdownTimer` (inside DungeonSheet and HUD badge)
- HUD variant: `hudLevel` pixel font (16px, PressStart2P) — format `HH:MM:SS`
- Sheet variant: `headingXl` (28px, Inter-Bold) — full legible large display
- Both use timestamp-based calculation from `startedAt` to survive backgrounding
- Source: `CONTEXT.md D-06`, `D-07`

### 4. Dungeon HUD Theme (HudScene mutation)
- During active session: pass `dungeonActive: boolean` prop to HudScene
- When `dungeonActive === true`: swap DayNightOverlay tint to dungeon shadow `rgba(0,0,0,0.65)`
- Stone wall treatment: an additional Skia Rect with dungeon stone surface `#1A1410` at 30% opacity drawn above background and below DayNightOverlay
- Torch elements: two Skia Circle shapes in amber `#C8621A` with a flicker animation (opacity 0.6-1.0, 400ms sine curve, offset phase on each torch)
- Character sprite is unchanged — the environment changes around them
- Source: `CONTEXT.md D-05`

### 5. `WelcomeBackToast`
- Rendered as RN View sibling outside Canvas (same pattern as FridayMessageBanner)
- Appears on app foreground if `detoxSession.status === 'active'`
- Shows for 3 seconds then fades out (300ms fade using Reanimated `withTiming`)
- Copy: "Still in the dungeon — [X]h [Y]m remaining" at `bodyMd`
- Background: `surface-800` with `border-default` border, `radius.lg` (12px)
- Source: `CONTEXT.md D-07`

### 6. `DungeonClearedFanfare` (full-screen overlay)
- Absolute-positioned View with semi-transparent backdrop `rgba(15,23,42,0.92)` (follows LevelUpOverlay pattern)
- Animated entry: ZoomIn spring (350ms, matches LevelUpOverlay)
- Heading: "Dungeon Cleared!" at `headingXl` (28px, Inter-Bold), color `colors.dark.xp` gold
- XP number: display-size number at `headingXl`, with a count-up animation (0 to earned XP over 800ms — `duration.celebration`)
- Mentor praise line: `bodyMd` (15px, Inter-Regular) in `textSecondary`, warm tone
- Haptic burst: 3x `impactAsync(Heavy)` at 0ms/100ms/180ms (matches LevelUpOverlay pattern)
- "Continue" button: full-width, accent fill — never auto-dismiss
- Source: `CONTEXT.md D-12`

### 7. `DetoxShieldBadge` (habit card augmentation)
- Small badge rendered inside HabitCard when `detoxSession.status === 'active'`
- Shield pixel icon (or text label "PROTECTED") at `hudLabel` size (10px, PressStart2P)
- Color: accent `#0D7C3D` with `rgba(13,124,61,0.15)` fill pill
- Only shown on habits whose completion window overlaps with detox session time
- Source: `CONTEXT.md D-13`, `D-14`

### 8. `EarlyExitConfirmation` (inline dialog / modal)
- Standard RN Modal or inline View overlay (not bottom sheet — needs clear foreground)
- Title: `headingMd` — "Leave the Dungeon Early?"
- Body: `bodyMd` — compassionate mentor voice copy (see Copywriting Contract below)
- XP penalty line: `bodyMd` with error color `#9B1B30` for the penalty amount
- Two buttons: "Keep Going" (accent fill, primary) + "Exit Anyway" (destructive outline)
- "Keep Going" is the visually dominant choice — right position, accent color
- Source: `CONTEXT.md D-09`, `D-10`

## Interaction Contracts

### Duration Chip Selection
- Tap chip: `duration.fast` (200ms) color transition on border + fill
- Only one chip selected at a time; selecting new chip deselects previous
- Disabled chip (deep variant already used): `onPress` blocked, opacity 0.4, no visual feedback

### Active Session Timer
- Timer updates every second via `setInterval` referencing wall-clock delta from `startedAt`
- On app foreground: recalculate immediately from stored `startedAt` (no "resume" animation needed — just accuracy)
- Timer display does not animate per tick — numbers update in place, no Reanimated per second (performance)

### Dungeon Door Glow
- Idle (no session): door icon renders at 100% opacity, no glow
- Tappable with no session: emerald glow ring at 60% opacity, static
- Active session: glow ring pulses (opacity 0.4 to 1.0, 2000ms cycle, `Easing.inOut(Easing.ease)`)
- Tap always opens DungeonSheet regardless of session state

### Fanfare Transition
- HudScene dungeon theme fades OUT over `duration.slow` (500ms) when fanfare appears
- Fanfare ZoomIn spring plays simultaneously with HUD fade
- After "Continue" tap: HudScene normal theme fades IN over `duration.normal` (300ms)

### Early Exit Penalty Preview
- Penalty XP value shown in real-time on the DungeonSheet "Exit Early" button label: "Exit Early (−X XP)"
- On tap: EarlyExitConfirmation appears — penalty recalculated at moment of tap (pro-rated from `startedAt` to `now`)

### Retry Behavior
- After early exit: DungeonSheet shows "Try Again" state for same-day retry
- Second entry does not reset penalty from first attempt — it is additive
- Once daily retry used: duration chips remain visible but CTA changes to "Available Tomorrow" (disabled state, `textMuted`)

### Reduced Motion
- All Reanimated `withTiming`/`withSpring` calls must check `AccessibilityInfo.isReduceMotionEnabled` at component mount
- If reduce motion enabled: skip torch flicker animation, skip ZoomIn fanfare (use instant opacity 1), skip XP count-up (show final value immediately)

## Accessibility Contract

Following ui-ux-pro-max CRITICAL rules:

| Rule | Implementation |
|------|---------------|
| `touch-target-size` | All duration chips min 44x44px; dungeon door min 44x44 hit slop; all buttons min 44px height |
| `aria-labels` | `accessibilityLabel` on dungeon door icon, shield badge, timer display |
| `focus-states` | All Pressable elements must have `android_ripple` or border highlight on focus |
| `color-contrast` | All body text on dark surfaces uses `textPrimary` (#F1F5F9) — passes 4.5:1 on `surface-900` (#0F172A) |
| `reduced-motion` | Check `AccessibilityInfo.isReduceMotionEnabled` — disable torch flicker, fanfare zoom, XP count-up |
| `loading-buttons` | "Enter the Dungeon" CTA disabled while session creation is in progress |

## Motion Contract

Source: `src/tokens/motion.ts` — pre-established, use token values.

| Interaction | Duration | Easing | Notes |
|-------------|----------|--------|-------|
| Chip select transition | 200ms (`fast`) | `easeOut` | Border + fill color change |
| Dungeon door glow pulse | 2000ms (`scene`) | `easeInOut` sine | Opacity 0.4 → 1.0 loop |
| Torch flicker | 400ms | `easeInOut` sine, offset per torch | Opacity 0.6 → 1.0, disabled on reduced motion |
| WelcomeBackToast fade-in | 300ms (`normal`) | `easeOut` | Auto-dismiss after 3s |
| WelcomeBackToast fade-out | 300ms (`normal`) | `easeIn` | Triggered at 3s mark |
| Fanfare ZoomIn spring | 350ms | `springify()` damping 15 stiffness 150 | Matches LevelUpOverlay |
| XP count-up | 800ms (`celebration`) | linear | 0 to earned XP; skip on reduced motion |
| HUD dungeon fade | 500ms (`slow`) | `easeInOut` | Normal → dungeon and reverse |
| Button press feedback | 100ms (`instant`) | — | `activeOpacity: 0.8` on Pressable |
