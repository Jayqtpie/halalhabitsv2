## AccountNudgeBanner — Interaction Contract

Source: `src/components/auth/AccountNudgeBanner.tsx` (built in Phase 7)

### Trigger Condition

All three conditions must be true simultaneously:

1. `!isAuthenticated` (user is a guest — authStore.isAuthenticated === false)
2. `!nudgeDismissed` (user has not previously dismissed — authStore.nudgeDismissed === false)
3. `firstUnlockedTitle !== null` (titleRepo.getUserTitles(userId) returns at least one entry)

Title name resolution: Use TITLE_SEED_DATA static lookup (avoids extra DB query). If
resolution fails, use generic fallback "The Steadfast" — never show "You've earned ."

### Mount Point

Render inside `app/(tabs)/index.tsx` (Home HUD screen), as the last child of the root
View. The component has `position: 'absolute', bottom: 0` in its own styles — it
floats above all HUD content without pushing layout.

### Animation Spec

| Event | Direction | Duration | Easing |
|-------|-----------|----------|--------|
| Enter (mount) | translateY +80 → 0 | 250ms | Easing.out(Easing.ease) |
| Exit (dismiss) | translateY 0 → +80 | 200ms | Easing.in(Easing.ease) |

Dismiss callback fires after 210ms delay (exit animation + 10ms buffer) via setTimeout.
This matches the established pattern in Phase 7 (07-CONTEXT.md: "AccountNudgeBanner
exit animation uses setTimeout(210ms) delay before calling setNudgeDismissed").

### Banner Layout

```
┌─────────────────────────────────────────────┐
│ 3px │  "Your progress is safe — for now."   [✕]
│ bar │  "You've earned {titleName}. Create a  │
│(FF) │   free account to back up your         │
│     │   discipline across devices."          │
│     │  [Keep My Progress]                    │
└─────────────────────────────────────────────┘
```

- Left accent bar: 3px wide, full height, `colors.dark.mercy` (#FFB347)
- Top border: 1px, `rgba(255, 179, 71, 0.3)` (mercy tint at 30% opacity)
- Background: `colors.dark.surface` (#1E293B)
- Dismiss button: top-right, 44×44px, `hitSlop: 8`
- CTA button: `alignSelf: 'flex-start'`, minHeight 44px, borderRadius 8, horizontal
  padding 16px, vertical padding 8px

### Self-Managed Guard

The component internally reads `isAuthenticated` and `nudgeDismissed` from authStore
and returns null if either is true. The parent (index.tsx) still guards render to
prevent unnecessary mounts — both guards run.

## SyncStatusIcon — State Contract

Source: `src/components/sync/SyncStatusIcon.tsx` (built in Phase 7)

Phase 8 does not change SyncStatusIcon code. Wiring the sync queue activates the
'syncing' and 'synced' states that were previously unreachable (queue was always empty).

| State | Condition | Icon Color | Badge | Caption |
|-------|-----------|------------|-------|---------|
| idle (offline/no account) | `!isAuthenticated \|\| syncStatus === 'idle'` | `#334155` (surface-700) | ✕ | "Offline" |
| syncing | `syncStatus === 'syncing'` | `colors.dark.primary` (#0D7C3D) | ↻ animated | "Syncing..." |
| synced | `syncStatus === 'synced'` | `colors.dark.primary` (#0D7C3D) | ✓ | "Synced" |
| error | `syncStatus === 'error'` | `colors.dark.xp` (#FFD700) | ! | "Sync failed — tap to retry" |

Touch target: 44×44px minimum, `hitSlop: 4`.
Tap behavior: triggers `flushQueue()` only in error state. Other states: no-op.

## Copywriting Contract

Source: `AccountNudgeBanner.tsx` (Phase 7 — copy is locked, do not change)

| Element | Copy |
|---------|------|
| Primary CTA | "Keep My Progress" |
| Banner heading | "Your progress is safe — for now." |
| Banner body | "You've earned {titleName}. Create a free account to back up your discipline across devices." |
| Dismiss accessibility label | "Dismiss banner" |
| CTA accessibility label | "Keep My Progress — create a free account" |
| SyncStatusIcon: offline | "Offline — no account connected" (accessibilityLabel) / "Offline" (caption) |
| SyncStatusIcon: syncing | "Sync in progress" (accessibilityLabel) / "Syncing..." (caption) |
| SyncStatusIcon: synced | "Synced" |
| SyncStatusIcon: error | "Sync failed, tap to retry" (accessibilityLabel) / "Sync failed — tap to retry" (caption) |

Empty state heading: Not applicable — Phase 8 has no new screens or list views.

Error state: SyncStatusIcon error caption ("Sync failed — tap to retry") is the only
user-visible error in this phase. It is action-oriented (tap to retry) and avoids
shame language. No modal or alert is shown for sync failure — silent retry via tap.

Destructive actions: None in Phase 8. No data deletion, account deletion, or
destructive confirmations introduced.

Adab compliance: All copy follows adab safety rails — no shame language, no guilt
framing. "Your progress is safe — for now." uses gentle urgency (not threat).
"Keep My Progress" is benefit-framed (not fear-based).

| Requirement | Implementation |
|-------------|----------------|
| Touch targets | 44×44px minimum on all interactive elements (AccountNudgeBanner CTA, dismiss button, SyncStatusIcon) |
| accessibilityRole | `"button"` on all Pressable elements |
