## Typography

All from `src/tokens/typography.ts`. This phase uses exactly 4 roles:

| Role | Token | Size | Family | Weight | Line Height | Letter Spacing |
|------|-------|------|--------|--------|-------------|----------------|
| Body (quest text, hadith body) | `typography.bodyMd` | 15px | Inter-Regular | 400 | 22px | 0 |
| Label (section headers, badge text, attribution) | `typography.bodySm` | 13px | Inter-Regular | 400 | 18px | 0 |
| Caption (progress count, deadline text, source citation) | `typography.caption` | 11px | Inter-Regular | 400 | 16px | 0.5 |
| HUD Label (Friday badge "2x XP" pixel text) | `typography.hudLabel` | 10px | PressStart2P | 700 | 14px | 1 |

Heading role: `typography.headingMd` (20px, Inter-SemiBold, weight 600, line-height 28px) is used only for the Al-Kahf quest card title "Surah Al-Kahf Challenge".

**Weight rule:** Two weights only — 400 (Inter-Regular) for all body/label/caption, 700 (Inter-Bold / PressStart2P) for HUD pixel labels and the completion state "Jumu'ah Honored!" text.

All from `src/tokens/colors.ts` `colors.dark` semantic set. App is dark-mode only.

| Role | Hex | Token | Usage |
|------|-----|-------|-------|

| Accent (10%) | `#0D7C3D` | `colors.dark.primary` | Specific elements only — see reserved list below |
| XP / Reward Gold | `#FFD700` | `colors.dark.xp` | Friday 2x badge label, XP reward text, Al-Kahf completion XP |
| Destructive | `#9B1B30` | `colors.dark.error` | Not used in this phase — no destructive actions |

**Accent (`#0D7C3D`) reserved for:**
- Friday badge border glow on HudStatBar chip
- Al-Kahf quest card border when active (same pattern as QuestCard `cardCompleted` border)
- Al-Kahf progress bar fill
- Jumu'ah completion checkmark circle fill
- "Mark Complete" button background on Al-Kahf card

**Gold (`#FFD700`) reserved for:**
- "2x XP" text inside the Friday HudStatBar badge chip
- XP reward badge on Al-Kahf quest card (+100 XP)
- Friday message hadith attribution text color (subtle gold tint to signal sacred source)
- XP float label on Jumu'ah habit completion (reuses existing `XPFloatLabel` pattern)

**Jumu'ah tag color:** `palette['emerald-400']` (`#34D399`) — the same warm emerald used for `shieldText` in HabitCard — applied to the "Jumu'ah" badge that appears on the Dhuhr slot on Fridays.

Accent reserved for list summary: Friday HUD chip border, Al-Kahf card border + progress fill, Jumu'ah completion circle, Mark Complete button. Do NOT use accent for the Friday message banner background or for the Jumu'ah tag badge (use emerald-400 instead to distinguish the tag from primary actions).

## Component Inventory

New components this phase introduces. All are React Native + Reanimated (no Skia canvas — all layered as absolute Views over the canvas per project pattern established in HudStatBar).

### 1. `FridayBoostBadge` (HudStatBar extension)

- **Location:** Conditionally inserted into `HudStatBar` row when `isFriday()` returns true
- **Layout:** Horizontal chip, flexShrink: 0. Sits between `BestStreakDisplay` and its right divider
- **Chip anatomy:** Gold pixel label "2×XP" (`typography.hudLabel`, `colors.dark.xp`) inside a container with `backgroundColor: 'rgba(13,124,61,0.15)'`, border `colors.dark.primary` width 1, `borderRadius: radius.sm` (4px), `paddingHorizontal: 6px`, `paddingVertical: 2px`
- **Animation:** Fade-in on mount with `Animated.timing` duration 300ms, opacity 0 → 1. On Friday end (midnight): fade out over 600ms, then show inline "Friday boost ended" text for 2 seconds (same typography.bodySm, color `colors.dark.textSecondary`), then unmount
- **Accessibility:** `accessibilityLabel="Jumu'ah 2x XP boost active"` on the chip View

### 2. `FridayMessageBanner` (HudScene overlay)

- **Location:** Absolute-positioned View layered over `HudScene` canvas, inside the Home screen's View hierarchy (NOT inside the Skia Canvas)
- **Trigger:** Shown on first Friday app open only; replaced by a condensed `FridayMessagePill` on subsequent opens same day
- **Layout:** Positioned at `top: safeAreaTop + 16px`, `left: 16px`, `right: 16px`
- **Background:** `colors.dark.hud.overlayBg`, border `colors.dark.hud.overlayBorder`, `borderRadius: radius.xl` (16px), `padding: 16px`
- **Content:** Hadith message text (`typography.bodyMd`, `colors.dark.textPrimary`) + attribution line below (`typography.caption`, `colors.dark.xp`, e.g., "(Muslim 854)")
- **Dismiss:** Tap anywhere on banner to dismiss; auto-dismisses after 8 seconds with fade-out animation (Reanimated `withTiming` 400ms)
- **Subsequent opens same day:** Condensed single-line `FridayMessagePill` — same styling but `paddingVertical: 8px`, message truncated to 1 line with attribution inline in parentheses
- **Accessibility:** `accessibilityRole="alert"` on first-open banner; `accessibilityLiveRegion="polite"` on pill

### 3. `AlKahfQuestCard` (QuestBoard extension)

- **Location:** Inserted as the first card in the Quest Board on Fridays, above existing quest sections. Uses `D-08` (extra card, does not replace existing slots)
- **Layout:** Extends existing `QuestCard` pattern (`colors.dark.surface`, `radius.md`, `spacing.md` padding, `spacing.sm` bottom margin)
- **Header row:** "Surah Al-Kahf Challenge" title (`typography.headingMd`, `colors.dark.textPrimary`), XP badge "+100 XP" (`typography.caption`, `colors.dark.xp`, badge border `colors.dark.primary`)
- **Deadline row:** Below header — clock icon (use "⏱" glyph at 12px) + "Complete by Maghrib · [HH:MM]" text (`typography.caption`, `colors.dark.textSecondary`)
- **Progress section:** Segmented progress bar showing 18 sections (Surah Al-Kahf has 110 ayat grouped into 18 logical sections for this UX). Each segment is a 4px-tall tick mark. Progress fill: `colors.dark.primary`. Progress count text: "X / 18 sections" (`typography.caption`, `colors.dark.textSecondary`)
- **Section tap area:** Below progress bar, a horizontally scrollable row of 18 numbered chips (28×28px, `borderRadius: radius.sm`). Completed chips: `backgroundColor: colors.dark.primary`, text `colors.dark.textPrimary`. Incomplete chips: `backgroundColor: colors.dark.surface`, border `colors.dark.border`
- **Active card border:** `colors.dark.primary` border width 1, same glow shadow as `QuestCard.cardCompleted`: `shadowColor: colors.dark.primary`, `shadowOpacity: 0.25`, `shadowRadius: 8`
- **Completed state:** Full QuestCard completed pattern — checkmark + "Jumu'ah honored! +100 XP" text (`typography.bodySm`, weight 600, `colors.dark.primary`)
- **Expired state (after Maghrib, incomplete):** Dimmed to `opacity: 0.45`, "Quest expired at Maghrib" text in `colors.dark.textMuted`, XP badge grayed out (`colors.dark.textMuted`)
- **Accessibility:** `accessibilityRole="none"` on card; section chip taps: `accessibilityRole="checkbox"`, `accessibilityLabel="Section {n}, {completed/incomplete}"`

### 4. `JumuahToggle` (HabitCard Dhuhr slot extension)

- **Trigger:** Appears below the standard Dhuhr HabitCard entry only on Fridays (`isFriday()` check in the habits list render)
- **Anatomy:** A secondary row beneath the Dhuhr card — NOT a separate card. Rendered as an inline View appended to the Dhuhr HabitCard footer area
- **Layout:** `flexDirection: 'row'`, `alignItems: 'center'`, `paddingHorizontal: spacing.md`, `paddingVertical: spacing.sm`, `backgroundColor: 'rgba(52,211,153,0.08)'` (faint emerald tint to signal Friday-only feature)
- **Badge:** "Jumu'ah" tag pill — `backgroundColor: 'rgba(52,211,153,0.15)'`, border `palette['emerald-400']` width 1, `borderRadius: radius.sm`, text `palette['emerald-400']` at `typography.caption`
- **Label:** "Attended Jumu'ah prayer today?" (`typography.bodySm`, `colors.dark.textPrimary`)
- **Toggle:** Custom tap-to-toggle circle (same 32×32 pattern as HabitCard checkCircle). Untoggled: `borderColor: colors.dark.textMuted`, hollow. Toggled: `backgroundColor: palette['emerald-400']`, `borderColor: palette['emerald-400']`, checkmark `colors.dark.textPrimary`
- **On toggle:** Triggers `expo-haptics` `ImpactFeedbackStyle.Light` + Skia animation frame in HudScene (mosque/minaret moment — see Animation spec below)
- **Dhuhr label behavior:** Dhuhr slot keeps its "Dhuhr" name label. The "Jumu'ah" badge appears as an inline chip to the right of "Dhuhr" text within the HabitCard `centerContent` area (no rename, per D-18)
- **Visibility:** Entire `JumuahToggle` row only renders on Fridays (`isFriday()` gate in component render). Zero height on non-Fridays — use conditional rendering, not visibility hidden
- **Accessibility:** Toggle: `accessibilityRole="checkbox"`, `accessibilityLabel="Jumu'ah prayer attended"`, `accessibilityState={{ checked: isToggled }}`

## Animation Contracts

### Friday Boost Badge Fade-In
- Library: `react-native-reanimated` (project standard)
- Pattern: `withTiming(1, { duration: 300 })` opacity on mount
- Easing: `Easing.out(Easing.ease)`
- No layout animation — conditional render only

### Friday Boost End Fade-Out
- Duration: 600ms fade, then display "Friday boost ended" text for 2000ms, then unmount
- Copy: "Friday boost ended" — `typography.bodySm`, `colors.dark.textSecondary`, same chip position
- After 2000ms: unmount the chip entirely (not hidden)

### Friday Message Banner Dismiss
- Auto-dismiss: `withTiming(0, { duration: 400 })` opacity after 8000ms delay
- Tap dismiss: same 400ms fade immediately
- No slide animation — opacity only (keeps HUD world undisturbed)

### Jumu'ah Completion Animation (Skia / HudScene)
- Trigger: `JumuahToggle` toggled ON
- Implementation: Single Skia frame drawn as an absolute View over `HudScene` (NOT inside Canvas — consistent with HudStatBar pattern)
- Visual: Minaret/mosque pixel art sprite rendered at screen center, scale-in from 0.6 → 1.0 over 400ms, hold 600ms, fade out over 300ms
- Asset: `assets/icons/jumuah-moment.png` (new pixel art asset, 64×64px, to be created with same pixel art style as existing habit icons)
- Mentor copy: Displayed as a `FridayMessagePill` style banner below the animation: mentor voice line (e.g., "Jumu'ah honored. May your Friday be blessed.") — `typography.bodyMd`, `colors.dark.textPrimary`, `colors.dark.hud.overlayBg` background, auto-dismiss after 3000ms
- Haptic: `expo-haptics` `NotificationFeedbackType.Success` on toggle ON

### Al-Kahf Section Progress Fill
- Each section chip tap: animate the chip from incomplete to complete state — `withSpring` backgroundColor transition not possible in RN; use opacity 0.4 → 1.0 on the fill overlay with `withTiming` 200ms
- Progress bar fill: `width` interpolation via `Animated.Value`, not layout animation

## Screen-by-Screen Contract

### Home Screen (HUD)

The Home screen renders `HudScene` (Skia Canvas) + `HudStatBar` (absolute View overlay at bottom) + `FridayMessageBanner` (absolute View overlay at top, Fridays only).

**Friday state (isFriday = true):**
1. `FridayBoostBadge` chip inserted into `HudStatBar` row between `BestStreakDisplay` divider and right divider
2. `FridayMessageBanner` shown on first open (top overlay, auto-dismiss 8s)
3. `FridayMessagePill` shown on return opens (condensed, top overlay)

**Non-Friday state:**
- No Friday chip, no banner. HudStatBar row unchanged.

**Boost-end transition (midnight Friday → Saturday):**
- `FridayBoostBadge` fades out over 600ms
- "Friday boost ended" text appears in place for 2000ms
- Component unmounts

### Habits Screen (Habit List)

**Friday state, Dhuhr habit present:**
1. "Jumu'ah" badge chip appears inline within the Dhuhr `HabitCard` centerContent, to the right of habit name
2. `JumuahToggle` row appears below the Dhuhr card in the list (appended within HabitCard footer, not as a sibling card)

**Non-Friday state:**
- Dhuhr card renders normally, no Jumu'ah references visible

### Quest Board Screen

**Friday state:**
1. `AlKahfQuestCard` appears as first card at top of quest board, above existing daily/weekly quest sections
2. Card shows active state with emerald border until completed or Maghrib
3. After Maghrib + incomplete: expired state (dimmed, neutral copy)
4. After completion: completed trophy state

**Non-Friday state:**
- `AlKahfQuestCard` not rendered. Quest board unchanged.
