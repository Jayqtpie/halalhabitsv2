## Spacing Scale

Declared values (all multiples of 4) — sourced from `src/tokens/spacing.ts`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, cloud icon margin |
| sm | 8px | Compact element spacing, list item gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, modal padding |
| xl | 32px | Layout gaps, screen-level horizontal padding |
| xxl | 48px | Major section breaks |

Component-level presets (sourced from `src/tokens/spacing.ts`):

| Token | Value | Usage |
|-------|-------|-------|
| cardPadding | 16px | Auth form card inner padding |
| buttonPaddingVertical | 12px | Sign In / Create Account buttons |
| buttonPaddingHorizontal | 24px | Sign In / Create Account buttons |
| modalPadding | 24px | Account creation modal, sign-out confirmation |
| modalElementGap | 16px | Spacing between modal form fields |

Exceptions:

- Touch targets for all interactive elements: minimum 44x44px (per ui-ux-pro-max `touch-target-size` rule). Cloud sync icon, dismiss button, and sign-in form inputs must meet this threshold.
- Account nudge banner: 12px vertical padding (hudCardPadding) to match HUD card aesthetic.

## Typography

Sourced from `src/tokens/typography.ts`. Two font families in use: Inter (body/UI copy) and Press Start 2P (game/pixel headings).

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|

- Pixel font (Press Start 2P) used for screen titles and section headers only — matching established phase convention.

## Color

Sourced from `src/tokens/colors.ts` (dark theme — dark-only for v1 per Phase 02 decision).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0F172A (surface-900) | Screen background, auth screen backdrop |
| Secondary (30%) | #1E293B (surface-800) | Auth form card surface, settings rows, account section card |
| Accent (10%) | #0D7C3D (emerald-500 / primary) | See reserved-for list below |
| Destructive | #9B1B30 (ruby-500 / error) | Delete account action only |

Accent (#0D7C3D) reserved for:

1. "Sign In" primary button background
2. "Create Account" primary button background
3. "Sync Now" button (manual sync trigger in settings)
4. Progress indicator / spinner fill when sync is in progress
5. Border highlight on focused form input fields (borderFocus token)
6. Success checkmark on cloud sync icon (synced state)

Second semantic color — Mercy Orange (#FFB347) reserved for:

- Account nudge banner background tint — the milestone nudge ("You earned [title]!") uses the mercy color token to signal warmth and positive encouragement, not warning.

Warning/info use gold (#FFD700 / xp token):

- Sync warning state on cloud icon (failed sync indicator uses gold-500, not ruby, to avoid alarm)

Destructive (#9B1B30) reserved for:

- "Delete Account" row label text
- "Delete Account" confirmation button background

## Interaction States

These are the unique interaction states Phase 7 introduces. All follow established React Native patterns from prior phases.

### Cloud Sync Status Icon

Located in the Settings screen header area (top-right of account section).

| State | Visual |
|-------|--------|
| Synced | Cloud icon + green checkmark, color: emerald-500, caption: "Synced" |
| Syncing | Cloud icon + animated spinner (Reanimated rotation, 300ms), color: emerald-500 |
| Sync failed | Cloud icon + warning badge, color: gold-500, caption: "Sync failed — tap to retry" |
| Offline / no account | Cloud icon with x, color: surface-700 (muted), no caption |

Touch target: 44x44px minimum. Tap triggers manual sync when in failed state.

### Auth Form Inputs

- Default border: #1E293B (border-default token)
- Focused border: #0D7C3D (borderFocus token), 2px width
- Error state: ruby-500 border + bodySm error copy directly below field (not toast)
- Password field: show/hide toggle icon at 24x24px within field, touch target padded to 44x44px

### Account Nudge Banner

Appears as a non-blocking bottom banner after first level-up or title unlock when user has no account.

- Background: mercy orange (#FFB347) at 15% opacity overlay on surface-800
- Left accent bar: 3px solid mercy orange
- Body copy: bodyMd Inter-Regular
- Title: "Your progress is safe — for now." (headingMd Press Start 2P)
- CTA button: accent emerald, "Keep My Progress"
- Dismiss: "x" icon top-right, 44x44px touch target, no confirmation required
- Never reappears after dismissed (settingsStore flag)

### Sign-Out

Non-destructive. Accessible from Settings > Account.

- Row label: "Sign Out" in textPrimary
- No confirmation dialog — sign-out is reversible (local data retained)
- After sign-out: settings screen reloads to guest state, no animation

### Delete Account

Destructive. Accessible from Settings > Account > Delete Account.

- Row label: "Delete Account" in ruby-500 (error token)
- Two-step confirmation: sheet modal with warning copy + explicit "Delete Everything" button in ruby-500
- "Delete Everything" button requires deliberate tap (not swipe)
- On completion: navigate to onboarding Welcome screen

## Screen Layout Notes

### Auth Screens (Sign In / Create Account)

- Full-screen layout, no tab bar visible
- Background: surface-900 (backgroundDeep)
- Centered card (surface-800) with cardPadding (16px), border-radius matches established card radius token
- Logo / pixel heading at top of card
- Form fields stack vertically with modalElementGap (16px) between them
- Primary CTA button at bottom of card, full width
- Guest mode link below card: bodySm, textSecondary, "Continue Without Account"
- Keyboard-avoiding behavior: `KeyboardAvoidingView` wrapping form

### Settings Screen — Account Section (additive to existing settings)

- Inserted as a new section at the top of the existing Settings screen
- Same row pattern as existing settings rows (listItemPaddingVertical: 12px, listItemPaddingHorizontal: 16px)
- SyncStatusIcon appears inline in section header row (right-aligned)
- Account section hidden / shows guest state when signed out

### Account Nudge Banner

- Positioned as a bottom sheet overlay, `position: absolute`, bottom: 0
- Does not block tab bar — renders above content, below tab bar z-index
- Animated in with Reanimated translateY from +80 to 0 over 250ms, ease-out
- Auto-dismissed via swipe-down gesture (Reanimated pan handler)

## Animation and Motion

Phase 7 introduces minimal new animation. All follow established Reanimated patterns from prior phases.

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Account nudge banner enter | translateY from +80px to 0 | 250ms | ease-out |
| Account nudge banner exit | translateY from 0 to +80px | 200ms | ease-in |
| Sync spinner | rotate 0 to 360 (looping) | 800ms per revolution | linear |
| Delete confirmation sheet enter | standard bottom sheet slide | 300ms | ease-out |
| Auth form error shake | translateX oscillation ±4px | 300ms | spring (damping 10) |

No celebration overlays for auth events — account creation is functional, not a game moment. Emotional reward happens at the milestone that triggered the nudge, not at account creation itself.

## Accessibility

Per ui-ux-pro-max skill `touch-target-size` and `color-contrast` rules:

- All touch targets: minimum 44x44px (sign-in/sign-out/delete rows, cloud icon, dismiss x)
- Form inputs: `accessibilityLabel` on each field, `returnKeyType` set ("next" for email, "done" for password)
- Password show/hide toggle: `accessibilityLabel="Show password"` / `"Hide password"` (not icon-only)
- "Delete Everything" confirmation: `accessibilityRole="button"` + `accessibilityHint="Permanently deletes your account and all data"`
- Error messages rendered as text below fields (not toast-only) so screen readers pick them up
- SyncStatusIcon: `accessibilityLabel` reflects current state ("Synced", "Sync in progress", "Sync failed, tap to retry")
- Color is not the only indicator: delete action uses both ruby color AND "Delete Everything" label text
