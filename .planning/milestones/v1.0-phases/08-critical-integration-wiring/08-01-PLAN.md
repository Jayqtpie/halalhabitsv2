After each write operation's `return` statement, add a fire-and-forget enqueue block. The pattern (insert example):

```typescript
async create(data: NewHabit) {
  const db = getDb();
  const result = await db.insert(habits).values(data).returning();

  // Enqueue for sync (non-blocking, skip for guests)
  try {
    const { isAuthenticated } = useAuthStore.getState();
    if (isAuthenticated) {
      assertSyncable('habits');
      syncQueueRepo.enqueue('habits', data.id, 'INSERT', data).catch(() => {});
    }
  } catch { /* enqueue must never block local write */ }

  return result;
}
```

For UPDATE operations, fetch the full updated row first to pass as payload:

```typescript
async update(id: string, data: Partial<Omit<NewHabit, 'id'>>) {
  const db = getDb();
  const result = await db
    .update(habits)
    .set({ ...data, updatedAt: new Date().toISOString() })
    .where(eq(habits.id, id));

  try {
    const { isAuthenticated } = useAuthStore.getState();
    if (isAuthenticated) {
      assertSyncable('habits');
      const updated = await db.select().from(habits).where(eq(habits.id, id));
      if (updated[0]) {
        syncQueueRepo.enqueue('habits', id, 'UPDATE', updated[0]).catch(() => {});
      }
    }
  } catch {}

  return result;
}
```

- `updateProgressAtomic()` -> UPDATE, payload: use the `.returning()` result directly
- `complete()` -> UPDATE, payload: re-fetched row after status change

**titleRepo.ts**:
- `grantTitle()` (entityType: 'user_titles') -> INSERT, payload: `data` (NewUserTitle)
- `seedTitles()` -> SKIP (static seed data, not user data)

- `upsert()` -> INSERT or UPDATE depending on `existing` check; after the write, re-fetch and enqueue with operation 'UPDATE' (upsert is always safe as UPDATE for sync)

**userRepo.ts** (entityType: 'users'):
- `create()` -> INSERT, payload: `data`
- `updateXP()` -> UPDATE, payload: re-fetched row
- `setActiveTitle()` -> UPDATE, payload: re-fetched row

Important rules:
- Use `useAuthStore.getState()` (static accessor), NOT the hook form — repos have no React context
- `.catch(() => {})` on every enqueue call — fire-and-forget
- Wrap entire enqueue block in try/catch that swallows — defense against any unexpected throw
- `assertSyncable()` call validates the table name is SYNCABLE before enqueue
