**The Discipline Metaphor:**
- The core metaphor: the player is on a journey of self-discipline, and their game world reflects their real-world consistency
- This is NOT a fantasy RPG -- the "world" is the player's own growth visualized through pixel art environments
- The metaphor must never imply spiritual measurement -- it reflects effort and consistency, not piety

**HUD Environments (3-4 progression tiers):**
Describe each environment that the Home HUD displays as the player levels up. Per user decision: "HUD is a game world scene (mosque courtyard or study room) that evolves as player levels up." Define:
- Environment 1 (Level 1-10): Starting environment -- describe the pixel art scene, what elements are present, the mood/lighting, colors from the jewel tone palette
- Environment 2 (Level 11-25): Evolved environment -- what changes, what new elements appear, how it feels different
- Environment 3 (Level 26-50): Advanced environment -- significant visual upgrade, richer details
- Environment 4 (Level 51-100): Mastery environment -- the aspirational final form
For each: describe the scene in enough detail that a pixel artist could create it. Include time-of-day variations (matches actual prayer times if possible).

One-paragraph descriptions of Nafs boss types (e.g., "The Procrastinator," "The Distractor," "The Doubter"). Mark clearly as v2. These exist for narrative coherence -- titles and quests may reference them thematically even in v1.

**blueprint/05-feature-systems.md** (BLUE-05):

Write detailed feature system specs. Each feature needs: purpose, rules, states, transitions, edge cases, and copy tone.

**V1 Feature Systems (full specs):**

1. **Habit Forge** -- The habit creation and management system
   - Preset Islamic habits library: list all presets (5 salah, Quran reading, dhikr, fasting, dua, tahajjud, sadaqah, etc.)
   - Custom habit creation: name (3-50 chars), frequency (daily/weekly/custom), optional time window, difficulty tier (affects base XP)
   - Habit states: active, paused, archived
   - Edit rules: what can be changed mid-streak vs what requires streak reset
   - Edge cases: what happens if user creates duplicate habit, max habit limit, habit reactivation after archive

2. **Quest Board** -- Daily and weekly challenge system
   - Quest types: daily (easy, 25-50 XP), weekly (medium, 100-200 XP), stretch (hard, 300-500 XP)
   - Rotation: how quests are selected/generated, variety mechanisms
   - Quest examples: "Complete all 5 salah today" (daily), "Maintain a 7-day Quran streak" (weekly), "Complete 50 habit check-ins this week" (stretch)
   - Quest states: available, in-progress, completed, expired
   - Edge cases: quest expires mid-progress, all quests completed early, timezone boundaries

3. **Salah Streak Shield** -- Prayer-time-aware streak protection
   - How it works: completing salah within its time window protects the salah streak for that prayer
   - Integration with adhan-js prayer time calculations
   - Edge cases: travel (shortened prayers), timezone changes, missing one salah but completing others

4. **Mercy Mode** -- Compassionate recovery system
   - Trigger conditions: habit streak breaks (zero completions for a habit in its frequency window)
   - Recovery quest generation: tailored to the broken habit, achievable in 2-3 days
   - Recovery rewards: partial streak credit, XP bonus for recovery completion
   - Copy tone: blended Islamic wisdom + game action. Provide 5 example messages.
   - States: inactive, active (streak broken), recovery-in-progress, recovered
   - Edge cases: multiple habits break simultaneously, recovery quest fails, user ignores Mercy Mode for days
   - Per user decision: "Mercy Mode ships WITH streaks (same phase, never separate)"

5. **Identity Titles** -- Achievement and identity system
   - Reference title table from blueprint/04-worldbuilding.md
   - How titles are displayed: current title shown on profile and HUD
   - Title selection: user can choose which earned title to display
   - Notification on unlock: celebration moment with title-specific copy
   - Title progress tracking: "You're 12 days away from The Steadfast"

6. **Muhasabah** -- Nightly reflection ritual
   - Prompt structure: 2-3 quick prompts (e.g., "What went well today?", "What's one thing you'd improve?", "Set one intention for tomorrow")
   - Duration target: 30-60 seconds, not journaling
   - Privacy: ALWAYS device-only, never synced (per adab safety rails)
   - Skippable: always, no penalty, no shame copy
   - XP: 10 XP bonus (gentle nudge, not coercive)
   - Edge cases: user opens at 11:59 PM, user already completed today, no habits completed today (different prompts?)

**V2 Feature Systems (one-paragraph summaries for economy coherence):**

7. **Nafs Boss Arena** -- Boss battles representing personal nafs struggles. Mark as v2. Note XP values for economy modeling (boss defeat: 200-500 XP). One paragraph on concept.

8. **Dopamine Detox Dungeon** -- Anti-doomscrolling challenge. Mark as v2. Note XP values. One paragraph.

9. **Friday Power-Ups** -- Jumu'ah-specific bonuses (2x XP on Friday completions). Mark as v2. Note economy impact.

10. **Accountability Duos** -- Private two-person mutual accountability. Mark as v2. One paragraph on privacy considerations.

11. **Barakah Economy** -- Cosmetic shop system. Mark as v2. Note currency model.
