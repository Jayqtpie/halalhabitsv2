**blueprint/07-screen-specs.md** (BLUE-07):

Write the screen-by-screen product specification. This is the largest and most detailed document in the blueprint. Per user decision: "Full wireframe-level screen specs with ASCII mockups."

Use this template for EVERY screen:
```
## [Screen Name]
### Purpose: [one sentence]
### Entry Points: [how user gets here]
### Layout: [ASCII mockup using box-drawing characters]
### Components: [table: name | type | behavior]
### Interactions: [tap/swipe/long-press actions]
### Animations: [trigger, description, duration, easing]
### Copy Examples: [element: "exact text string"]
### Edge States: [table: state | display | copy]
```

**Screens to spec (12+ required, aim for 14-15):**

1. **Onboarding - Welcome** -- First screen, app personality introduction, "Begin your journey" CTA
2. **Onboarding - Niyyah** -- Intention setting screen ("What discipline are you building?"), free-text or guided prompts
3. **Onboarding - Habit Selection** -- Pick from preset Islamic habits, toggle on/off, see XP preview
4. **Home HUD** -- The crown jewel. Pixel art game world scene showing: level, XP bar, streak count, active quests, today's progress. Per user decision: "full pixel immersion." This is the most detailed spec.
5. **Habits List** -- Daily checklist view. Modern UI with pixel accents. Shows each habit with completion checkbox, streak count, time window (for salah). Grouped by time of day or category.
6. **Habit Detail** -- Individual habit stats: streak history (calendar/heatmap), total completions, XP earned, edit/archive actions
7. **Habit Create/Edit** -- Form: name, frequency, time window, difficulty tier. Preset selection or custom.
8. **Quest Board** -- Grid/list of available quests. Daily/weekly/stretch tabs or sections. Each quest shows: description, XP reward, progress bar, time remaining.
9. **Profile** -- Current title (large, prominent), level + XP bar, streak stats, earned titles grid, achievement highlights
10. **Settings** -- Sections: notifications, prayer calculation method, privacy controls, data export/delete, dark mode, about
11. **Muhasabah** -- Nightly reflection. Card-based prompts, swipe through 2-3 questions, optional intention for tomorrow, XP earned indicator, skip button always visible
12. **Mercy Mode Overlay** -- Appears on streak break. Compassionate message, recovery quest details, "Begin Recovery" CTA. NOT a separate screen -- an overlay/modal on Home HUD.
13. **Level Up Celebration** -- Full-screen modal on level up. New level number, XP animation, any unlocks (title, environment, quest tier), "Continue" CTA
14. **Title Unlock** -- Modal showing earned title with rarity badge, flavor text, "Equip Title" / "Keep Current" options

For the Home HUD, create a detailed ASCII mockup showing the pixel art scene area, the overlay UI elements (level, XP bar, streak), and the interaction zones. This is the most important wireframe in the entire blueprint.

Edge states required for EVERY screen: empty state, loading state, error state, offline state, plus domain-specific states (first-time, streak-broken, etc.).

All copy examples must use wise mentor voice and pass adab safety rails. Use actual text strings, not placeholders.

Add cross-reference links to sections 05, 06, and 08.
