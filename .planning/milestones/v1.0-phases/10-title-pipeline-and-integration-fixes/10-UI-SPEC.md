## Phase UI Character

Phase 10 is a wiring phase. It has zero new screens and zero new UI components. The four
code fixes (mercyRecoveries query, muhasabahStreak query, your-data userId hook, quest
partial-progress parameter) are invisible at the pixel level — users see no new layouts,
no new navigation, and no copy changes except that previously unreachable title unlock
celebrations now correctly trigger. This spec exists to lock the contract for those
existing surfaces that Phase 10 touches: `your-data.tsx` and the shared title-unlock
celebration overlay.

## Spacing Scale

Source: `src/tokens/spacing.ts` — all multiples of 4. No exceptions for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, list item gap |
| md | 16px | Card padding, list item horizontal padding, default element spacing |
| lg | 24px | Section padding, modal padding |
| xl | 32px | Layout gaps |
| xxl | 48px | Major section breaks |

Component presets (from `src/tokens/spacing.ts` `componentSpacing`):

| Preset | Value | Note |
|--------|-------|------|
| cardPadding | 16px | 4-unit value |
| buttonPaddingVertical | 12px | 3-unit value — intentional for button ergonomics; reviewed and approved |
| buttonPaddingHorizontal | 24px | 6-unit value |

## Screen Inventory

Phase 10 touches exactly two existing screens. No new screens or modals are introduced.

### 1. your-data.tsx (Modified)

**Change:** Replace module-scope `const USER_ID = 'default-user'` with
`const userId = useAuthStore((s) => s.userId)` inside the component body.

**Visual delta:** Zero. The screen layout, copy, color, and component tree are
unchanged. The userId flows invisibly into the export and delete service calls.

**Focal point:** The "Export My Data" primary action button (emerald fill, top of action
group) is the visual anchor of the screen. The "Delete All Data" button (ruby fill, below
export) reads as secondary-destructive — same size, lower visual weight due to ruby vs
emerald contrast against the dark surface.

**Interaction contract (unchanged):**
- Export: Pressable "Export My Data" → async → ActivityIndicator replaces label during
  load → Alert.alert on success/failure
- Delete: Pressable "Delete All Data" → Alert.alert confirmation with two-step
  confirmation → async → ActivityIndicator → navigation reset to onboarding on success
- Back navigation: router.back() on header back arrow

**Accessibility (unchanged):**
- "Delete All Data" Pressable: minimum 44px height, ruby fill, white text
- ActivityIndicator replaces button label during async operations (loading-buttons rule)
- No new interactive elements introduced

### 2. Title Unlock Celebration Overlay (Triggered, not modified)

**Change:** The existing `TitleUnlockOverlay` component now fires for Mercy Mode and
Muhasabah titles that were previously unreachable because `checkTitles` returned no
match due to hardcoded 0 values.

**Visual delta:** Zero to the component itself. Users now see the overlay for two
previously-broken title categories.

**Interaction contract (unchanged):**
- Overlay appears as StyleSheet.absoluteFillObject over current screen
- Displays title name, rarity badge, and glow animation
- User taps anywhere to dismiss
- XP float text appears in gold (#FFD700) rising from completion point

**Rarity glow contracts for newly-enabled titles:**
- Mercy Mode titles (e.g., "The Merciful"): Mercy orange glow (#FFB347) — uses
  `colors.dark.mercy` token, consistent with Mercy Mode branding throughout app
- Muhasabah titles (e.g., "The Reflective"): Sapphire glow (#1B3A6B) — uses
  `colors.dark.secondary`, consistent with reflection/introspection palette

## Copywriting Contract

Phase 10 introduces no new user-visible copy. All copy below is pre-existing in the
codebase. Listed here as the contract baseline for the surfaces this phase touches.

| Element | Copy | Source |
|---------|------|--------|
| your-data screen heading | "Your Data" | existing |
| Export CTA | "Export My Data" | existing |
| Export success | "Your data has been exported and shared." | existing |
| Export error | "Failed to export data. Please try again." | existing |
| Delete CTA | "Delete All Data" | existing |
| Delete confirmation title | "Delete All Data?" | existing |
| Delete confirmation body | "This will permanently delete all your habits, streaks, XP, and progress. This cannot be undone." | existing |
| Delete confirmation cancel | "Keep My Data" | updated — replaces generic "Cancel"; uses Alert.alert `text` property |
| Delete confirmation confirm | "Delete Everything" | existing |
| Delete success | App navigates to onboarding — no copy, screen transition | existing |

**Adab constraints (unchanged — enforced from CLAUDE.md):**
- No shame copy on delete: "Delete Everything" not "Lose All Progress"
- No guilt copy anywhere in your-data.tsx

- Dismiss button reads "Keep My Data" (affirmative, data-safe) paired with "Delete Everything" (explicit destructive)

**New title copy — scope clarification:** Title names and unlock messages are defined in
`TITLE_SEED_DATA` (src/db/seeds/title-seeds.ts). Phase 10 does not add new title
definitions. It only enables the existing Mercy Mode and Muhasabah title entries to
actually trigger. No copy contract change needed.
